# 倉儲部 AI 語音宣讀系統

上傳純文字檔（.txt），系統會自動處理數字唸法與破音字，產生一個可分享的語音宣讀頁面。宣讀端會逐句高亮、可調語速與語音，適合在會議中播放正式文件。

---

## 系統需求

- **Node.js 18 以上**（建議 20 LTS）。確認版本：`node -v`
- 現代瀏覽器（Chrome / Edge 建議）。語音朗讀使用瀏覽器內建的 Web Speech API，
  音色取決於使用者電腦安裝的語音，**建議播放端的電腦安裝中文（繁體）語音**以取得最佳效果。

---

## 快速開始（本機 / 內網）

在專案目錄下依序執行：

```bash
# 1. 安裝依賴（第一次執行，或依賴有更新時）
npm install

# 2. 啟動伺服器
npm start
```

啟動成功會看到：

```
🚀 倉儲部AI語音系統後端已成功啟動！
👉 後台管理網址：http://localhost:3000/index.html
```

接著用瀏覽器開啟 **http://localhost:3000/index.html** 即可使用。

若要讓**同一區網內的其他電腦**連入，把 `localhost` 換成這台主機的區網 IP
（例如 `http://192.168.1.50:3000/index.html`），並確認防火牆已放行 3000 埠。

若要在本機測試「後台密碼保護」，啟動時帶入環境變數即可：

```bash
# macOS / Linux
ADMIN_PASSWORD=你的密碼 npm start

# Windows（PowerShell）
$env:ADMIN_PASSWORD="你的密碼"; npm start
```

啟動後開後台頁會跳出帳號密碼視窗（帳號預設 `admin`）。不帶此變數則不啟用驗證（適合純內網）。

## 使用流程

1. 在後台頁（index.html）選擇一個 `.txt` 檔案上傳。
2. 系統回傳一條「分享連結」，並顯示宣讀預覽（可先試聽）。
   - **一般模式**：顯示分享連結，複製後傳給需要的人。
   - **播放模式**：上傳後直接開啟全螢幕宣讀頁，適合會議現場直接投影。
3. 對方開啟分享連結（listen.html）即可看到內文並逐句朗讀，可調整語音與語速。

分享連結預設保留 **7 天**，逾期自動清除（可於 `server.js` 的 `FILE_TTL_MS` 調整）。

## 破音字字典（dict.json）

系統會把「朗讀用文字」中的破音字替換成**同音的替身字**，讓語音引擎唸對，
而**畫面上顯示的文字完全不變**。例如字典寫 `"重量": "仲量"`，畫面仍顯示「重量」，
但朗讀時會唸成「仲量」的音。

### 兩種維護方式

1. **（推薦）透過後台 API**：系統提供 `GET /api/dict` 與 `POST /api/dict`，
   若有搭配管理介面可直接編輯存檔，格式一定正確、即時生效、不需重啟。
2. **直接編輯 dict.json 檔案**：存檔後即時生效。

### 撰寫規則

- 格式為 `"要替換的字詞": "替身字"`，例如 `"長官": "掌官"`。
- **詞一律優先於單字**：系統會自動依字數長到短排序，較長的詞先比對。
  所以可以先設單字的預設讀音，再針對例外的詞另加規則。例如：
  ```json
  {
    "行": "形",       // 「行」單獨出現時的預設讀音
    "銀行": "銀航",   // 「銀行」的「行」讀音不同，額外覆蓋
    "長官": "掌官"
  }
  ```
- ⚠️ **務必使用半形符號**：JSON 不接受全形空白或全形引號。
  用中文輸入法編輯時容易誤打全形空白（看不出來），會導致整份字典解析失敗、
  上傳時出現「伺服器處理檔案失敗」。若遇到此錯誤，優先檢查 dict.json 是否混入全形字元。

## 文字正規化（normalize.js）

朗讀前會自動把數字轉成中文唸法，並保護編碼類數字：

| 原文 | 朗讀 |
|------|------|
| `3,500 件` | 三千五百 件 |
| `12.5%` | 百分之十二點五 |
| `1,002,000 元` | 一百萬零二千 元 |
| `15.5 公斤` | 十五點五 公斤 |
| 料號 `A-1203-B` | 保留原樣（不亂唸） |
| 訂單 `20240715` | 逐位唸「二零二四零七一五」 |

規則說明：緊鄰英文字母或連字號的數字（料號、批號、型號）會原樣保留；
連續 6 位以上的純數字視為編號逐位唸；其餘數字轉為一般中文數量唸法。

## 對外部署：讓任何人有網址就能使用（GitHub + Render）

想讓不特定使用者從任何電腦連入，最省事的方式是把專案放上 GitHub，再用 Render
這類 PaaS 平台自動部署。**HTTPS 由平台自動提供**（這對本系統是必要的，見下方警告）。

### ⚠️ 重要：對外一定要用 HTTPS，否則「沒有聲音」

本系統的語音朗讀使用瀏覽器的 Web Speech API，而瀏覽器規定這個 API **只在 HTTPS
（或 localhost）下才能運作**。如果你用「http:// + 公開 IP」對外，頁面打得開、文字看得到，
但**按播放不會有任何聲音**。使用 Render / Railway 等平台會自動配置 HTTPS，可直接避開這個問題。

### 步驟

1. **上傳到 GitHub**
   在專案目錄執行（需先安裝 git、並在 GitHub 建好一個空 repo）：
   ```bash
   git init
   git add .
   git commit -m "initial commit"
   git branch -M main
   git remote add origin https://github.com/你的帳號/你的repo.git
   git push -u origin main
   ```
   `.gitignore` 已設定好，`node_modules`、`.env`、`filedb.json` 等不會被上傳。

2. **在 Render 建立服務**
   - 到 https://render.com 註冊並登入（可用 GitHub 帳號）。
   - New → Web Service → 連結你剛才的 GitHub repo。
   - Render 會自動偵測 `render.yaml`，或你手動填：
     - Build Command：`npm install`
     - Start Command：`npm start`

3. **設定後台密碼（務必）**
   在 Render 服務的 **Environment** 頁面新增環境變數：
   - `ADMIN_PASSWORD` = 你自訂的強密碼
   - （選填）`ADMIN_USER` = 後台帳號，預設 `admin`

   沒設 `ADMIN_PASSWORD` 的話，任何人都能上傳檔案、竄改字典 — 對外務必設定。

4. **完成**
   Render 會給你一個網址（如 `https://warehouse-tts.onrender.com`）。
   - 後台上傳頁：`.../index.html`（開啟時會跳出帳號密碼視窗）
   - 宣讀分享連結：上傳後自動產生，**公開可開，不需密碼**，可直接分享給任何人。

### ⚠️ 另一個重要限制：分享連結在免費方案上可能遺失

多數 PaaS 平台（含 Render 免費方案）的檔案系統是「暫時的」— 服務重新部署、或閒置一段時間
自動休眠後重啟，都會**清空** `filedb.json`，導致先前產生的分享連結全部失效。

因應方式：
- **短期／會議當下使用**：影響不大，當次上傳、當次分享、會議結束即可。
- **需要長期保存連結**：改用付費方案的持久磁碟（Render Disk），或把資料改存到外部資料庫
  （如免費的 Supabase / MongoDB Atlas）。這需要額外開發，需要時可再協助。

## 其他強化方向（非必要）

- **行程守護**（自管主機才需要）：用 `pm2` 或 systemd 讓伺服器崩潰自動重啟。
  例：`npm i -g pm2 && pm2 start server.js --name warehouse-tts`
- **資料庫**：目前用 JSON 檔持久化，資料量大或需長期保存時建議改用 SQLite 或雲端資料庫。
- **防濫用**：可再對上傳頻率做限制（目前單檔上限 10MB）。

## 常見問題

**Q：上傳後出現「伺服器處理檔案失敗」？**
A：最常見原因是 `dict.json` 格式錯誤（尤其混入全形空白）。用文字編輯器檢查該檔是否為合法 JSON。

**Q：朗讀沒有聲音，或音色很奇怪？**
A：Web Speech API 依賴使用者電腦安裝的語音。請在播放端電腦的作業系統設定裡安裝中文（繁體）語音，
   並在宣讀頁右上的「語音」下拉選擇合適的語音。

**Q：只支援哪些檔案？**
A：目前僅支援 `.txt` 純文字檔（自動辨識 UTF-8 / Big5 編碼）。

**Q：重啟伺服器後舊的分享連結會失效嗎？**
A：不會。已上傳的資料存於 `filedb.json`，重啟後自動讀回（除非已超過保留期限）。
